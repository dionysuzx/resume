I'm a passionate software engineer. I started my journey with computers in middle school where I'd take apart the hardware and install Linux distros from scratch; and I did the same with my Android phone, mainly an unlocked Galaxy Nexus. From high school is when I started programming and knew that I'd be doing software.

I search deeply for the right solution to solve a problem (see [my keyboard](https://github.com/dionysuzx/zmk-config)). I believe that it's very easy for an engineer to over-abstract or over-use "design principles", causing tight coupling and unnecessary maintenance in a codebase (because I've made this mistake). I'm particularly passionate about processes, especially developer processes. This is reflected throughout my entire career and is a topic I'm very passionate to talk about.

I'm still learning every day.
